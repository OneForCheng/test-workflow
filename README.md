# test-workflow

基于 Kiro Steering 的测试用例生成工作流。通过结构化的交互流程，将需求转化为高质量、可追溯的测试用例。

## 项目结构

```
.kiro/steering/test-case-generator.md   # 测试用例生成器（Steering 文件）
test-case-templates/                     # 测试用例模版目录
  └── default-template.md               # 默认通用模版
test-cases/                              # 生成的测试用例存放目录
测试用例模版.csv                          # 参考用的 CSV 模版样例
```

## 使用方式

1. 在 Kiro 对话中通过 `#` 引用 `test-case-generator` steering 文件
2. 按提示提供需求（支持对话描述、文件引用、Jira 卡号、外部链接等方式）
3. 按照七步流程逐步确认，最终生成测试用例文件

## 工作流程

1. **需求收集** — 提供需求输入
2. **需求确认** — 确认需求范围
3. **需求分析与澄清** — 消除歧义，可多轮
4. **模版选择与粒度确认** — 选择用例模版和详细程度
5. **输出格式确认** — 选择 `.md` 或 `.csv` 格式
6. **测试用例生成** — 按模版生成用例文件
7. **用例评审与调整** — 确认并调整用例

## 需求输入方式

| 方式 | 说明 |
|------|------|
| 对话描述 | 直接在对话中描述需求 |
| 文件引用 | 通过 `#File` 引用需求文档 |
| Jira 卡号 | 提供卡号如 `PROJ-123`，通过 MCP 工具读取 |
| 外部链接 | 提供 Confluence、Notion 等链接 |

## 输出格式

- **Markdown**（`.md`）：结构清晰，便于阅读和版本管理
- **CSV**（`.csv`）：便于导入测试管理工具或 Excel 编辑

生成的文件命名格式：`TC-{模块名}-{YYYYMMDD}.md/.csv`

## 自定义模版

可在 `test-case-templates/` 目录下添加新的模版文件。生成用例时会提示选择使用哪个模版。
